# 失败与修复示例

成功示例展示理想结果；本目录说明知识库常见的坏状态以及怎样用最小改动修复。

## 单一来源被写成综合结论

**现象**：一篇论文的摘要被改写后放进 `wiki/synthesis/`，措辞却写成“研究普遍证明”。

**修复**：将内容移回对应 Source，保留样本与条件；等第二个独立来源出现后再提出 Synthesis 计划。

## 近义概念重复

**现象**：出现“间隔学习”“分散学习”“间隔效应”三个高度重叠页面。

**修复**：选择一个规范标题，把其他名称放入 `aliases`；合并非重复内容，更新反向链接，并在日志记录重命名映射。

## 新页面全部孤立

**现象**：摄取后生成十多个页面，但没有链接到来源、索引或其他概念。

**修复**：删除或合并没有复用价值的页面；其余页面至少链接一个 Source 和一个相关页面。

## 冲突被静默覆盖

**现象**：新会议记录改变了交付日期，旧决策页面被直接改写，看不到原决定。

**修复**：保留旧决定及生效时间，新增变更记录；并列链接两次会议来源，标明当前有效状态和确认人。

## 敏感信息进入共享库

**现象**：Source 页复制了真实姓名、邮箱、客户编号和本机绝对路径。

**修复**：停止同步；轮换可能泄露的凭据；一致脱敏；检查 Git 历史和附件；按 [隐私与安全实践](../../docs/privacy-and-security.md) 重新设置权限。

## 建议的修复顺序

1. 先停止自动写入。
2. 输出只读健康报告。
3. 优先处理隐私、覆盖和不可追溯结论。
4. 再处理断链、重复和孤立页。
5. 每批修复后重新运行检查，并向 `log.md` 追加记录。
