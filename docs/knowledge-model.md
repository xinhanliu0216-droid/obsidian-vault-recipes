# 知识对象与建页决策

本指南回答最容易让知识库失控的问题：一段新信息应该留在来源页，还是创建 Concept、Entity、Topic 或 Synthesis？

## 五类核心对象

| 类型 | 回答的问题 | 最低创建条件 | 不应创建的情况 |
|---|---|---|---|
| Source | 这一份材料说了什么？ | 对应一份可定位原始资料 | 没有原始文件或定位信息 |
| Concept | 这个稳定概念是什么？ | 能跨来源复用，定义边界清晰 | 只是某位作者的一次措辞 |
| Entity | 这个人物、组织或对象是谁？ | 在多个页面中反复出现并承担关系 | 只出现一次且没有复用价值 |
| Topic | 围绕这个问题有哪些信息？ | 持续变化、存在多个角度或待解决问题 | 只是一个稳定定义 |
| Synthesis | 多份来源合起来支持什么判断？ | 至少两个独立来源，写明证据与限制 | 只有单一来源或只是摘要拼接 |

## 决策顺序

1. **先建 Source。** 新材料首先得到忠实、可定位的来源页。
2. **先搜索再新建。** 检查现有标题、别名和近义页面，能更新就不重复创建。
3. **判断是否可复用。** 一次性细节留在 Source；稳定定义才进入 Concept。
4. **判断是对象还是问题。** 人、组织、产品等长期对象进入 Entity；围绕问题组织的信息进入 Topic。
5. **检查来源数量。** 只有跨至少两个独立来源形成判断，才进入 Synthesis。
6. **无法连接时暂缓。** 新页面至少链接一个已有页面；否则先在 `index.md` 标为待整理。

## 常见边界

### Concept 与 Topic

- “变异系数是什么”更适合 Concept。
- “什么时候不应比较标准差”更适合 Topic。
- 稳定定义用 Concept，围绕问题、条件或争议组织信息用 Topic。

### Source 与 Synthesis

- “论文 A 的实验发现”属于 Source。
- “论文 A 与论文 B 对延迟测验的共同证据及限制”属于 Synthesis。
- 把一份来源改写得更流畅，不会自动变成 Synthesis。

### Entity 与 Topic

- “某产品的基本身份与相关页面”属于 Entity。
- “某产品当前的增长问题”属于 Topic。

## 综合页的证据要求

综合页必须包含当前主张、支持来源、反对或冲突来源、不可比条件、当前未知、状态与最近核验日期。

证据冲突时不要投票式合并。先检查研究对象、时间范围、定义、测量口径和上下文是否一致，再决定能否综合。

## 最小原则

每次新建前问三个问题：

1. 以后会从多个地方链接到它吗？
2. 它是否比现有页面表达了不同职责？
3. 删除它后，是否会失去重要的可复用结构？

三个问题都答不上来时，优先更新 Source 或现有页面。
