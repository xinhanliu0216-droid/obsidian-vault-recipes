# 3B：学术研究 LLM Wiki

研究 Wiki 的核心不是收藏多少论文，而是让每个主张、方法和综合结论都能回到具体来源，并保留证据限制与冲突。本章以“间隔学习是否改善长期记忆”为例，演示论文摄取、文献集合处理和研究问答。

## 1. 这个场景解决什么问题

常见研究资料库问题：

- 只读摘要，却补写全文方法；
- 把作者在讨论中的解释写成实验结果；
- 不同对象、测量和时间范围的研究被直接合并；
- 只保留支持当前观点的论文；
- DOI、页码、样本量和效应值被生成或抄错；
- 新论文出现后直接覆盖旧综合，看不到证据变化。

研究场景的 `CLAUDE.md` 必须控制结论强度，并把“文献说什么”和“当前综合判断”分开。

## 2. 示例知识库

```text
raw/
├── research-question.md
├── paper-a.pdf
├── paper-b.pdf
├── paper-c-supplement.pdf
├── review-2025.pdf
└── screening-notes.md
```

研究问题：间隔学习是否改善大学生的长期记忆？

主要输出：论文来源页、方法与测量页、Claim/Evidence、冲突分析、阶段性 Synthesis 和研究空白。

## 3. 材料怎样组织

### 单篇论文

一篇正式摄取的论文对应一个 Source。PDF 很长时可按摘要、方法、结果、讨论分批提取，但仍保持同一论文身份。

### 文献集合

先建立 manifest：

```markdown
# 文献摄取清单

- 研究问题：间隔学习是否改善大学生长期记忆？
- 纳入范围：2015–2026，大学生，至少 7 天延迟测验

| 文献 | 状态 | 纳入 | 原因 | 正文 | 附录 | 下一步 |
|---|---|---|---|---|---|---|
| paper-a | done | yes | 符合对象与结局 | 已读 | 无 | 提取效应与限制 |
| paper-b | doing | pending | 延迟时间待核实 | 结果已读 | 未读 | 检查附录 |
| paper-c | pending |  |  |  |  |  |
```

重复文献、预印本与正式版本、勘误和补充材料都需要明确关系。

## 4. 页面模型和创建判断

### Claim

明确、可检验的主张，例如“在至少 7 天延迟后，间隔学习提高回忆表现”。必须链接提出或检验该主张的 Source。

### Method

用于跨文献比较的方法，例如“延迟测验”“组间随机实验”。只有能复用时才建页。

### Evidence

记录支持或反对某个 Claim 的具体结果，包含样本、对象、干预、对照、结局、时间、统计和不确定性。

### Debate

只有结论或关键前提真实冲突时创建。术语不同、对象不同或时间范围不同可能只是不可比，不一定是争议。

### Synthesis

至少两个独立来源。必须包括当前判断、支持证据、反证、不可比条件、偏倚、未知、范围和最近核验日期。

## 5. 摄取单篇论文

```text
请先读取 research-question.md、CLAUDE.md 和 paper-a.pdf。
不要直接形成综合结论。

请先给出计划：
1. 论文身份、版本和可解析范围；
2. 研究问题、样本、设计、干预、对照、结局与时间；
3. 准备创建的 Source；
4. 哪些 Method、Claim 或 Evidence 值得更新；
5. 哪些内容只是作者解释；
6. 哪些页、表、图或附录尚未覆盖。
```

确认后，Source 应区分：

- 研究实际测量到什么；
- 作者怎样解释；
- 研究限制是什么；
- 对当前研究问题能支持到什么程度。

## 6. 文献集合的分批处理

不要一次让 AI“读完 50 篇论文并给结论”。推荐流程：

1. 去重并确认版本；
2. 按纳入标准初筛；
3. 每批 3–5 篇，先完成 Source；
4. 统一对象、设计、结局和时间字段；
5. 每批更新 Claim/Evidence；
6. 记录排除理由和未获取全文；
7. 关键文献完成后才更新 Synthesis。

每批结束输入：

```text
请更新 manifest，列出本批纳入、排除、未确定和未解析内容。
比较本批与现有 Claim 的关系：支持、反对、不可比或不相关。
先提出 Synthesis 是否需要更新以及理由，等我确认后再写。
```

## 7. 常用查询提示词

### 是否证明

```text
根据当前 Wiki 回答：现有研究是否证明间隔学习普遍更好？
先说明研究设计能够支持到什么程度，再列支持证据、反证、不可比条件和未知。
不要使用未摄取论文或外部常识。
```

### 方法比较

```text
比较当前文献使用的延迟测验设计。
统一按对象、延迟时间、测量方式、样本量、优点、局限和相关 Source 输出。
不可比较的地方不要强行排序。
```

### 研究空白

```text
根据现有 Evidence 和 Synthesis 列出真正的证据空白。
区分：没有研究、研究结果冲突、设计不足、对象不匹配和资料尚未摄取。
```

## 8. CLAUDE.md 场景规则逐段解释

- **使用目标**：要求形成证据网络，不追求论文数量。
- **输入与处理单位**：规定一篇论文一个 Source，同时允许分段提取。
- **页面类型**：把 Claim、Method、Evidence、Debate 分开。
- **摄取规则**：强制拆分方法、结果、解释和限制。
- **综合规则**：防止对象和测量不同的研究被简单投票。
- **查询输出**：要求回答结论强度、反证和未知。
- **完成标准**：保证引用位置、文献范围和时间状态清晰。

完整文件：[研究 CLAUDE.md 样例](../../examples/claude-md/research-CLAUDE.md)。

## 9. 怎样改成你的研究领域

需要你决定：

- 研究问题和纳入排除标准；
- 证据等级或研究设计分类；
- 引用格式；
- 主要比较维度；
- 何时允许更新结论置信度；
- 预印本、灰色文献和非正式资料是否纳入。

交给 AI 的提示词：

```text
请根据下面的研究问题和纳入标准修改 research-CLAUDE.md：
{研究问题、对象、干预/暴露、对照、结局、时间、研究设计}

请只给最小 diff，并设计测试：
1. 摘要主张不能变成实验事实；
2. 不可比研究不能强行合并；
3. 未读附录必须进入遗漏清单。
暂时不要编辑。
```

## 10. 失败案例与修复

### 失败：摘要被写成结果

修复：Source 分设“实际结果”和“作者主张”，并要求结果链接表图或正文位置。

### 失败：多数论文支持，所以结论确定

修复：综合先检查样本、方法、结局和时间是否可比，不按论文数量投票。

### 失败：生成 DOI 或页码

修复：无法从材料确认的标识使用 `unverified`，不得补写。

### 失败：新论文静默覆盖旧结论

修复：Synthesis 保留版本、最近核验日期和更新理由，旧结论不消失。

## 11. 完成验收清单

- [ ] 每篇正式文献有 Source；
- [ ] 方法、结果、作者解释和限制分开；
- [ ] 数值保留样本、单位和位置；
- [ ] Claim 有提出和检验来源；
- [ ] Evidence 标明支持、反对或不可比；
- [ ] 纳入、排除、未获取和未解析资料有记录；
- [ ] Synthesis 至少使用两个独立来源；
- [ ] 综合写明范围、反证、未知和最近核验日期；
- [ ] 没有生成 DOI、页码或不存在的文献。
