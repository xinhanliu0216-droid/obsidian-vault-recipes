# 3A：课程学习 LLM Wiki

课程 Wiki 的目标不是“把课件都总结一遍”，而是把课程材料转化成能够解释、练习、回忆和复盘的知识网络。本章以“统计学课程库”为例，展示从建立规则到第一次摄取、长讲义处理、复习查询和规则修复的完整过程。

## 1. 这个场景解决什么问题

典型课程资料包括教材、课件、课堂记录、习题、参考答案和个人错题。如果只是把它们放进 Obsidian，会出现：

- 同一个概念在不同课次重复解释；
- AI 生成大量摘要，却不能用于复习；
- 课件中的一句话被误当成严格定义；
- 还没有答题，AI 就猜测“薄弱点”；
- 每道题都拆成页面，Wiki 很快失控；
- 教材、教师口径和参考资料冲突时没有记录。

课程场景的 `CLAUDE.md` 应让 Agent 围绕“理解—练习—反馈—复习”循环工作。

## 2. 示例知识库

假设你正在学习统计学，材料包括：

```text
raw/
├── syllabus.md                 # 教学大纲与考试范围
├── lesson-01-descriptive.md    # 第一课课堂记录
├── chapter-02.pdf              # 教材第二章
├── homework-01.md              # 作业题
├── homework-01-answer.md       # 参考答案
└── my-answer-01.md             # 个人作答
```

主要输出是：

- 能讲清楚的概念页；
- 主动回忆问题；
- 有代表性的例题和反例；
- 来自真实作答的错题复盘；
- 期中、期末专题综合。

## 3. 材料怎样组织

### 短材料

一节课堂记录、一次作业或一份大纲可以作为一个 Source。课堂记录需要日期和课程范围；作业题、参考答案、个人作答不要合并成同一个原始文件。

### 长材料

教材或长讲义先建立 `_staging/manifest.md`：

```markdown
# 教材摄取清单

- 版本：第 3 版
- 目标范围：第 1–8 章
- 考试范围：第 1–6 章

| 批次 | 范围 | 状态 | Source | 遗漏 | 下一步 |
|---|---|---|---|---|---|
| 01 | 第 1 章 | done | [[chapter-01]] | 图 1-3 未解析 | 第 2 章 2.1–2.3 |
| 02 | 第 2 章 2.1–2.3 | doing |  |  |  |
| 03 | 第 2 章 2.4–2.7 | pending |  |  |  |
```

按章或教学单元分批，不按固定字数任意切断公式和例题。

## 4. 页面模型和创建判断

### Concept

用于跨课次复用的核心概念，例如“方差”“标准差”“抽样分布”。至少包含定义、直觉、条件、例子和常见误用。

不应建页：只在一道题里出现的一次性数字、老师的临时提醒、没有解释价值的符号。

### Question

用于主动回忆，例如“为什么不能直接比较不同量纲变量的标准差？”答案必须链接课程来源。

不应建页：把每个小标题机械改成问题，或者答案只是复制一句原文。

### Example

用于展示概念、边界或迁移。普通重复练习留在 Source；只有具有代表性、反例性或能连接多个概念的题才建页。

### Mistake

必须来自真实作答。记录：我怎样错、为什么错、正确理解、以后怎样检查。

不应建页：AI 根据常识预先生成“你可能会错的地方”。

### Synthesis

用于跨章节框架、概念比较和考试专题，例如“离散程度指标如何选择”。必须建立在已处理章节上，不能根据目录猜测未读内容。

## 5. 第一次摄取完整演示

把 `lesson-01-descriptive.md` 放入 `raw/` 后输入：

```text
请读取 CLAUDE.md、index.md 和 raw/lesson-01-descriptive.md。
先不要写文件。

请给出摄取计划：
1. 这份课次覆盖哪些概念；
2. 准备创建哪个 Source；
3. 哪些内容值得更新现有 Concept；
4. 是否有内容值得形成 Question 或 Example；
5. 哪些细节只留在 Source；
6. 完成后怎样验证。
```

合理计划可能是：

```text
创建：wiki/sources/lesson-01-descriptive.md
创建：wiki/concepts/方差与标准差.md
创建：wiki/topics/Q-为什么不同量纲不直接比较标准差.md
更新：index.md
追加：log.md
不创建：每个示例数字的独立页面
```

用户确认后执行。最终 Source 应保留原文件路径和课次范围；Concept 不复制整份课堂记录，而是链接 Source；Question 的答案能回到来源。

## 6. 从作业形成真实错题

不要只摄取参考答案。把作业题、参考答案和个人答案同时提供：

```text
请比较 homework-01.md、homework-01-answer.md 和 my-answer-01.md。
先只报告：
- 哪些题确实答错；
- 错误属于计算、概念、条件遗漏还是表达问题；
- 哪些错误值得建立 Mistake 页；
- 会更新哪些 Concept 或 Question。
不要根据未作答题目推断我的薄弱点。
```

Mistake 页应链接题目、个人作答和依据，不把一次错误写成“长期能力不足”。

## 7. 长教材的分批演示

第一次处理 `chapter-02.pdf`：

```text
请先盘点 chapter-02.pdf，不总结整章。
建立或更新 _staging/manifest.md，说明：
- 版本与总页数；
- 小节结构；
- 公式、图表和例题是否可解析；
- 建议批次；
- 第一批准备覆盖的页码和小节。
等我确认批次后再摄取。
```

每批结束要求 Agent 输出：

- 实际覆盖页码和小节；
- 新建或更新的页面；
- OCR、图表和公式遗漏；
- 疑似与已有概念重复的内容；
- 下一批起点。

只有考试范围内关键章节完成后，才能建立期中或期末 Synthesis。

## 8. 常用查询提示词

### 主动回忆

```text
根据已摄取的第 1–2 章生成 8 个主动回忆问题。
先只显示问题；我回答后再逐题给答案、证据页面和纠正建议。
不要使用未摄取章节。
```

### 概念讲解

```text
根据当前 Wiki 解释“标准差”。
按直觉、正式定义、符号与单位、适用条件、例子、反例和常见误用组织。
每个重要判断链接 Source；课程材料没有覆盖的部分明确说明。
```

### 考试复习

```text
根据 syllabus.md 和已摄取内容制作复习清单。
区分：已覆盖、需要练习、存在真实错误、尚未摄取。
不要把阅读过等同于已经掌握。
```

## 9. CLAUDE.md 场景规则逐段解释

- **使用目标**：约束 Agent 生成可练习内容，而不只是摘要。
- **输入与处理单位**：防止教材、课件、作业和个人答案混为一类来源。
- **页面类型**：定义 Concept、Question、Example、Mistake 和 Synthesis 的职责。
- **创建条件**：控制页面数量，禁止制造假错题和空洞问题。
- **摄取规则**：要求先 Source、后知识页，并为公式保留条件。
- **查询输出**：规定复习时先提问、后显示答案。
- **完成标准**：用于验收来源、范围和真实掌握证据。

完整文件：[课程 CLAUDE.md 样例](../../examples/claude-md/course-CLAUDE.md)。

## 10. 怎样改成你自己的课程

需要由你决定：

- 考试形式和范围；
- 教材、教师和通用知识冲突时谁优先；
- 是否需要公式推导；
- 错题按题目、概念还是错误类型归档；
- 是否采用间隔复习；
- 什么表现算“掌握”。

可以把答案交给 AI：

```text
请根据下面的课程情况修改 course-CLAUDE.md：
{课程、考试、材料和复习目标}

只提出最小 diff。说明每条修改解决什么问题，并设计三个测试：
1. 摄取一节课；
2. 从真实作答形成 Mistake；
3. 在未覆盖章节上拒绝过度回答。
暂时不要编辑文件。
```

## 11. 失败案例与修复

### 失败：生成大量 Question

根因：只定义了 Question 格式，没有创建门槛。

修复：增加“必须可反复练习、答案有证据、不能只是标题改写”。

### 失败：把教师提醒写成严格事实

根因：没有区分课程口径和一般知识。

修复：Source 保留教师原话；Concept 标明“本课程口径”，冲突时并列来源。

### 失败：根据阅读次数判断掌握

根因：把接触证据当成表现证据。

修复：薄弱点只根据真实作答、错题和用户自评判断。

## 12. 完成验收清单

- [ ] `raw/` 未被修改；
- [ ] 每份正式材料有 Source；
- [ ] Concept 能回到课程来源；
- [ ] Question 有可验证答案；
- [ ] Mistake 来自真实错误；
- [ ] 教材长文档有 manifest 和恢复点；
- [ ] 考试范围与未覆盖内容明确；
- [ ] index 已更新，log 只追加；
- [ ] 没有把未作答或未阅读内容写成掌握结论。
