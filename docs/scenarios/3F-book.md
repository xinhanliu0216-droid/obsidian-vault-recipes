# 3F：图书与长材料 LLM Wiki

图书场景是长文档摄取的代表。目标不是让 AI 一次“读完并总结”，而是建立可恢复、可核验的阅读流程：章节有 Source，概念和论点逐步连接，跨章节综合只覆盖实际处理范围。

## 1. 这个场景解决什么问题

- 没读全文却生成全书主旨；
- 把一章观点写成作者最终立场；
- 摘录没有页码，无法回查；
- OCR 缺页、图表和脚注被当成已处理；
- 每个人名、案例和术语都建孤立页面；
- 中断后不知道处理到哪里，只能从头开始；
- 不同版本页码混用；
- 跨章节主题过早确定，后文反例被忽略。

图书 `CLAUDE.md` 应把“盘点—分批—Source—知识页—跨章综合”写成明确协议。

## 2. 示例知识库

```text
raw/
├── book.pdf
├── book-metadata.md
├── author-interview.md
└── reading-purpose.md
```

阅读目的：理解作者如何论证“练习比重复阅读更能形成长期记忆”，并为课程学习建立可引用的概念与论点。

## 3. 材料怎样组织

图书和长报告先按版本、目录、章节、附录和可解析性组织。原始文件保持完整，处理进度与解析产物进入 _staging/。

### 首轮盘点

第一次不要总结，输入：

```text
请盘点 raw/book.pdf，不总结全书。
请报告：
- 书名、作者、版本和总页数；
- 目录与章节；
- PDF 是文本、扫描还是混合格式；
- 哪些页、图表、脚注或附录可能无法解析；
- 建议的语义批次；
- manifest 应记录哪些字段。
```

建立 `_staging/manifest.md`：

```markdown
# 图书摄取清单

- 书名：示例
- 版本：中文版第 2 版
- 总页数：360
- 阅读目的：分析学习方法的证据与论证

| 批次 | 范围 | 状态 | Source | 解析遗漏 | 新知识页 | 下一起点 |
|---|---|---|---|---|---|---|
| 01 | 前言 + 第 1 章，1–38 页 | done | [[chapter-01]] | 图 1-2 | [[检索练习]] | 第 2 章 39 页 |
| 02 | 第 2 章，39–72 页 | doing |  | 表 2-1 待核 |  |  |
| 03 | 第 3 章，73–110 页 | pending |  |  |  |  |
```

## 4. 页面模型和创建判断

### Chapter / Source

每章或明确页码范围建立 Source，记录覆盖范围、摘要、主要论点、关键位置和未解析内容。

### Argument

记录作者提出的主要论点、依据、限定和反驳对象。必须能回到章节和页码。

### Concept

跨章节复用、边界相对稳定的概念。不要为只出现一次的术语建页。

### Character / Case

只有跨章节持续发挥作用的人物、案例或对象才建页。一次性例子留在 Chapter Source。

### Theme

跨章节逐步发展的主题。阅读中使用 tentative 状态，后续章节可能修改。

### Synthesis

用于跨已处理章节比较论点、主题和证据。必须声明版本、覆盖章节和未读范围。

## 5. 摄取第一批

```text
请摄取 manifest 中批次 01：前言和第 1 章。
先给计划，不要写文件：
1. 实际可读取页码；
2. Chapter Source 的结构；
3. 可能更新的 Concept、Argument 或 Theme；
4. 哪些案例只留在 Source；
5. 图表、脚注和 OCR 遗漏；
6. 批次完成后的恢复点。
```

用户确认后执行。批次结束必须更新 manifest 和 log，不能只生成笔记。

## 6. 中断与恢复

下一次会话先输入：

```text
请读取 CLAUDE.md、index.md、log.md 和 _staging/manifest.md。
告诉我上次处理到哪里、有哪些遗漏、下一批是什么。
不要从头重读已经完成的章节，也不要修改文件。
```

恢复点应足以让新 Agent 接手：版本、最后页码、已建页面、疑似重复、遗漏和下一步都明确。

## 7. 跨章节综合的门槛

例如要建立“作者如何看待重复阅读”Synthesis，至少检查：

- 涉及该主题的章节是否已处理；
- 每个关键论点有 Chapter Source；
- 前后章节是否发生立场变化；
- 作者主张与引用研究是否分开；
- 是否存在反例、限定或未解析图表；
- 未读章节是否可能影响结论。

如果全书只处理了前三章，应写“基于第 1–3 章的阶段性综合”，不能写“全书认为”。

## 8. 常用查询提示词

### 章节回顾

```text
根据 Chapter Source 回顾第 2 章。
按本章问题、主要论点、证据、概念、例子、限制和与前章关系组织。
每项保留页码；未解析表图单列。
```

### 论点追踪

```text
追踪“检索练习”在已处理章节中的变化。
按章节列作者主张、依据、限定和反例，不使用未读章节。
```

### 全书问题

```text
基于当前已处理范围，作者如何论证练习优于重复阅读？
先声明版本、已覆盖和未覆盖章节，再给阶段性结论、证据和未知。
```

### 引文查找

```text
查找与“必要困难”相关的直接引文。
只返回已核验原文、页码、章节和上下文；转述不要放在直接引文栏。
```

## 9. CLAUDE.md 场景规则逐段解释

- **目标**：规定阅读目的，避免无目标摘要。
- **输入单位**：要求先确认版本、目录和解析状态。
- **页面类型**：区分 Chapter、Argument、Concept、Theme 和 Synthesis。
- **长文档协议**：使用 manifest、语义批次和恢复点。
- **引用规则**：要求引文、转述和页码对应实际版本。
- **查询方式**：先声明覆盖范围，再回答。
- **完成标准**：检查章节、遗漏、恢复点和跨章综合门槛。

完整文件：[图书 CLAUDE.md 样例](../../examples/claude-md/book-CLAUDE.md)。

## 10. 怎样改成你的长材料

需要你决定：

- 阅读目的是学习、研究、写作还是检索；
- 使用哪个版本；
- 批次按章、小节、日期还是文档组成；
- 位置使用页码、电子位置还是段落编号；
- 是否需要人物、案例、引文、论点或术语索引；
- 什么范围完成后允许跨章节综合；
- 是否允许参考作者访谈或外部资料。

交给 AI：

```text
根据这份材料的目录、版本和阅读目的修改 book-CLAUDE.md。
请先给最小 diff，并设计测试：
1. 首轮只能盘点，不能总结全书；
2. 中断后能从 manifest 恢复；
3. 未读章节不能进入全书结论；
4. 引文必须匹配版本和页码。
```

## 11. 失败案例与修复

- **一章等于全书主旨**：Synthesis 必须写覆盖范围，未完成前标阶段性。
- **缺失页被忽略**：manifest 增加解析遗漏列，完成条件要求清零或解释。
- **每个人名都建页**：只有跨章节复用才创建 Entity/Character。
- **中断后重复处理**：每批记录下一起点和已建页面。
- **页码对不上**：在 Source 记录版本，不混用不同版本位置。
- **直接引文被改写**：直接引文与转述分栏，原文逐字核验。

## 12. 完成验收清单

- [ ] 版本、总页数、目录和阅读目的明确；
- [ ] manifest 覆盖所有章节和附件；
- [ ] 每批有范围、Source、遗漏和下一起点；
- [ ] Chapter Source 有准确页码范围；
- [ ] 引文匹配实际版本；
- [ ] 一次性人物和例子没有过度建页；
- [ ] Theme 在阅读中保持 tentative；
- [ ] Synthesis 明确已覆盖和未覆盖章节；
- [ ] 中断后可从恢复点继续；
- [ ] 没有补写未读内容。
