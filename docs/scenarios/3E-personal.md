# 3E：个人成长 LLM Wiki

个人 Wiki 可以帮助整理目标、习惯、日记和实验，但它最容易越过隐私与解释边界。Agent 只能整理记录、提出有限观察，不能根据少量文本定义人格、诊断健康状况或替代专业建议。

## 1. 这个场景解决什么问题

- 日记数量增加后难以回顾；
- 目标、习惯和复盘分散；
- 同一种问题反复出现，却没有证据化观察；
- AI 根据一天日记推断长期人格；
- 情绪被改写成客观事实；
- 私密原文被复制到共享综合页；
- AI 用羞辱性语言评价目标执行；
- 健康、心理、法律或财务问题被当成可以诊断。

个人场景的第一原则是：先确定什么可以被 Agent 读取和综合，再谈自动整理。

## 2. 示例知识库

```text
raw/
├── journal/2026-07-01.md
├── journal/2026-07-02.md
├── weekly-review-2026-W27.md
├── goals-2026-Q3.md
├── habit-log-sleep.csv
└── experiment-morning-writing.md
```

主要输出：Goal、Habit、Reflection、Experiment 和有证据/反例的阶段性 Synthesis。

## 3. 材料怎样组织

个人材料先按隐私等级和时间单位组织，再决定是否摄取。日记按日，复盘按周或月，目标和实验保留独立来源。

### 隐私分层

建立前先分级：

| 级别 | 示例 | 建议 |
|---|---|---|
| 普通 | 阅读、学习、公开目标 | 可正常摄取 |
| 私密 | 情绪、关系、工作复盘 | 限制读取和写回范围 |
| 高敏感 | 健康、财务、身份、客户信息 | 默认不进入共享 Wiki，必要时脱敏 |
| 禁止 | 密钥、账号、精确地址、他人隐私 | 不交给 Agent |

可以按目录设置更严格规则。综合页不应复制大段私密原文，只链接必要 Source 并使用最小信息。

## 4. 页面模型和创建判断

### Goal

包含目标、动机、衡量方式、期限、复查日期和状态依据。不能由 AI 自行设定目标。

### Habit

记录行为定义、触发条件和实际数据。“早睡”过于模糊，应明确为“23:30 前关灯”等可观察行为。

### Reflection

区分事件、感受、解释、需求和下一步。感受是真实体验，但不自动证明对他人动机的判断。

### Experiment

有假设、期限、观察指标、停止条件和复盘日期，例如“两周晨间写作实验”。

### Synthesis / Pattern

只有跨多次独立记录重复出现，并同时检查反例和缺失数据时，才能提出阶段性模式。使用“当前记录显示”而不是“你就是”。

## 5. 第一次摄取日记

```text
请读取 journal/2026-07-01.md，先不要写文件。
请区分：
- 发生的事件；
- 用户明确表达的感受；
- 用户自己的解释；
- Agent 可能作出的推断；
- 不应进入共享 Wiki 的敏感内容。

只提出 Source 和必要 Reflection 的计划，不建立长期 Pattern。
```

一篇日记不能证明长期模式。Source 可以忠实记录；Reflection 只整理本次复盘。

## 6. 从多周记录形成阶段性观察

```text
请根据 2026-W25 至 W28 的周复盘检查“睡眠不足与下午注意力下降”是否反复出现。
先只报告：
- 支持记录；
- 反例；
- 缺失数据；
- 其他可能解释；
- 是否达到建立阶段性 Synthesis 的条件。
不要进行医疗诊断。
```

如果只有两次主观记录，应使用低置信度并说明限制。若涉及持续健康风险，应建议咨询专业人士，而不是给诊断。

## 7. 个人记录的长周期处理

多年日记不应一次全部总结：

1. 在 manifest 列出日期范围和缺失区间；
2. 按周或月分批；
3. 每批先建立日期 Source/Review；
4. 记录仅本批观察，暂不推断长期人格；
5. 跨季度综合前检查反例和生活环境变化；
6. 明确哪些年份、主题或目录禁止综合。

## 8. 常用查询提示词

### 周复盘

```text
根据本周已允许读取的记录生成周复盘。
按事件、进展、阻碍、感受、已验证经验、待观察问题和下周实验组织。
不要诊断，不引用被标为私密且禁止写回的原文。
```

### 目标进度

```text
根据 Goal 页面和实际记录说明 Q3 目标进展。
区分有证据完成、正在进行、缺少记录和目标需要重新确认。
不要根据没有记录推断失败。
```

### 个人模式

```text
检查“拖延总发生在任务不明确时”这个假设。
列支持记录、反例、缺失、其他解释和当前置信度。
不要把阶段性观察写成人格标签。
```

## 9. CLAUDE.md 场景规则逐段解释

- **目标**：限定为整理和观察，不做诊断。
- **输入单位**：让日记、目标、数据和实验分别保留来源。
- **页面类型**：区分 Goal、Habit、Reflection、Experiment 和阶段性综合。
- **创建条件**：禁止一次事件升级为稳定模式。
- **解释规则**：分开用户原话、事实和推断。
- **查询方式**：要求反例、日期范围和非诊断措辞。
- **完成标准**：检查敏感信息和真实状态依据。

完整文件：[个人 CLAUDE.md 样例](../../examples/claude-md/personal-CLAUDE.md)。

## 10. 怎样改成你自己的个人 Wiki

需要你决定：

- 哪些目录允许 Agent 读取；
- 哪些内容禁止综合或共享；
- 日、周、月、季度的复盘周期；
- 目标和习惯怎样衡量；
- 什么证据足以提出 Pattern；
- 健康、财务和关系材料怎样处理；
- 是否允许写回原始日记（通常不允许）。

交给 AI：

```text
请根据我的隐私分级和复盘方式修改 personal-CLAUDE.md。
先只给 diff，并说明每条规则保护什么边界。
测试：
1. 单篇日记不能形成长期人格结论；
2. 私密原文不能进入共享综合；
3. 缺少记录不能被判断为目标失败。
```

## 11. 失败案例与修复

- **一天日记定义人格**：Pattern 至少跨多次记录并检查反例。
- **情绪变成事实**：区分“我感到被忽视”和“对方故意忽视”。
- **没有记录等于失败**：状态标为缺少证据，不作评价。
- **私密原文进入综合页**：使用最小必要转述并链接受限 Source。
- **生成健康诊断**：只整理症状记录与就医问题，明确非专业判断。

## 12. 完成验收清单

- [ ] Agent 只读取获准目录；
- [ ] 原始日记未被修改；
- [ ] 敏感信息没有进入不必要页面；
- [ ] 事件、感受、解释和推断分开；
- [ ] Pattern 有多次记录、反例和范围；
- [ ] Goal/Habit 状态来自实际证据；
- [ ] 没有医疗、心理、法律或人格诊断；
- [ ] 长周期材料有 manifest 和缺失区间；
- [ ] index/log 不暴露不必要隐私。
