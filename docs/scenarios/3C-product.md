# 3C：产品分析

产品场景需要把用户原话、归纳问题、机会假设、实验和决策分层。

## 相对通用模板的变化

- 增加 Feedback、Problem、Opportunity、Experiment、Decision。
- 单条反馈只能形成弱信号。
- Opportunity 默认是假设，并包含验证方法。
- 决策保留替代方案和反对意见。

## 使用

1. 复制 [产品规则](../../templates/scenarios/product.md)。
2. 定义用户群、数据来源和决策人。
3. 运行 [产品验收案例](../../tests/cases/product.md)。

## 典型失败

- 高频反馈被直接等同于高商业价值。
- 竞品营销文案被当成已验证事实。
- 会议中的想法被写成正式决策。
- 实验结论超出样本范围。
