# 第 4 章：修改和演化 CLAUDE.md

`CLAUDE.md` 是知识库的操作协议，不是愿望清单。你不需要亲自写出所有提示词，但必须向 AI 提供真实目标、材料、权限、输出和失败案例；具体规则措辞、diff 和测试可以交给 AI 起草。

完整的设计问题、长文档法则和提示词见 [如何设计和修改知识库的 CLAUDE.md](design-your-claude-md.md)。

## 1. 先描述使用方式，不要先堆规则

修改前回答：

1. 这个库最终支持什么行动或决定？
2. 实际输入有哪些格式、长度和定位信息？
3. 最常用的输出是什么？
4. 哪些内容不能修改、联网或共享？
5. 哪些知识值得独立成页？
6. Agent 已经发生过什么具体错误？
7. 怎样从文件结果判断任务完成？

AI 可以根据答案提出规则，但不能替你决定隐私边界、权威来源和证据门槛。

## 2. 规则应该放在哪里

| 内容 | 推荐位置 |
|---|---|
| 每次会话都适用的目标、目录、权限和证据边界 | 根 `CLAUDE.md` |
| 某个领域的页面类型、输入特点和输出格式 | 场景覆盖层 |
| 只对某类路径适用的约束 | 嵌套 `CLAUDE.md` 或 `.claude/rules/` |
| 长文档摄取等多步骤流程 | `_guides/` 或 Skill；根文件保留触发条件和硬边界 |
| 断链、字段等机械检查 | 脚本或 Hook |
| 个人路径和私有偏好 | 本地规则文件并加入 `.gitignore` |
| 只有一次的要求 | 当前对话 |

根文件建议保持在 200 行以内。简短不是删掉关键规则，而是把细节放到正确层次。

## 3. 从失败现象修改，而不是从想象修改

一次规则变更遵循：

1. 保存最小失败案例，例如“整本书一次总结并引用了未读章节”。
2. 找到根因：缺少触发条件、动作、边界，还是验证？
3. 判断修复应进入根规则、场景层、Skill 或脚本。
4. 让 AI 提出最小 diff 和测试，不直接编辑。
5. 确认后应用，并重跑原失败案例。
6. 删除被新规则替代的重复提醒。

## 4. 一条规则怎样写清楚

每条关键规则包含：

- **触发条件**：什么时候适用；
- **动作**：Agent 具体做什么；
- **边界**：禁止什么；
- **验证**：怎样知道正确完成。

示例：

> 当材料包含多个章节或超过一个会话可可靠核验的范围时，先创建 manifest 并按语义边界分批。每批记录覆盖范围、遗漏和下一起点。禁止在关键章节未处理时生成全书综合。完成后检查章节无遗漏或重复。

## 5. 设计场景覆盖层

场景文件不是另写一份完整 `CLAUDE.md`，而是回答通用规则无法回答的领域问题：

- 真实输入是什么，合理处理单位是什么；
- 哪些页面类型是领域特有的；
- 每类页面何时创建、何时不创建；
- 长材料按什么语义边界分批；
- 常用查询应该怎样组织；
- 哪些混淆最危险；
- 场景完成标准是什么。

跨场景时选一个主场景，只复制次场景真正需要的差异。规则冲突时明确优先级，不把两份文件机械拼接。

## 6. 长文档规则至少写清七件事

1. 怎样盘点目录、页数、附件和版本；
2. 批次按章、节、日期还是其他语义边界；
3. `_staging/manifest.md` 记录什么；
4. 中断后怎样从恢复点继续；
5. Source、知识页和 Synthesis 分别在何时创建；
6. 图表、脚注、OCR 失败和未读章节如何记录；
7. 什么条件满足后才允许全文综合。

如果这些问题没有答案，Agent 很容易把“能读取文件”误当作“已经可靠处理全文”。

## 7. 让 AI 帮你完善的提示词

```text
请审查当前 CLAUDE.md，并根据我的真实使用方式提出最小修改。

目标：{知识库支持的行动或决定}
输入：{材料类型、长度、结构、版本和定位信息}
输出：{最常用的查询或产物}
权限：{不能修改、联网、外传或自动执行的内容}
失败案例：{一个真实发生的问题}

请先不要编辑。请输出：
1. 问题根因；
2. 哪些内容应放根 CLAUDE.md、场景层、Skill 或脚本；
3. 页面创建和不创建条件；
4. 长文档分批、manifest、恢复点和综合门槛；
5. 最小 diff；
6. 三个测试提示及预期文件结果；
7. 仍需我决定的问题。
```

## 8. 版本记录

实质规则变化在 `log.md` 追加：

```markdown
## [YYYY-MM-DD] schema | 简短标题

- 症状：
- 根因：
- 变更：
- 测试：
- 结果：
- 被替代规则：
```

规则长期失效时先诊断根因，不要继续追加同义句。重复提醒会稀释真正重要的边界。
