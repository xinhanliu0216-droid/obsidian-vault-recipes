## 三层结构

- `raw/`：用户提供的原始资料。只读，不移动、不改名、不覆盖、不删除。
- `_staging/`：长文档解析产生的可重建中间文本、分块和清单。可以写入，但不能作为最终结论的唯一证据；重建或清理前先确认。
- `wiki/`：Agent 维护的 Markdown 知识层。
  - `wiki/sources/`：每份来源的摘要和定位信息。
  - `wiki/entities/`：人物、组织、产品、项目等实体。
  - `wiki/concepts/`：可复用概念与定义。
  - `wiki/topics/`：跨来源主题。
  - `wiki/synthesis/`：综合、比较与阶段性结论。
- `index.md`：内容索引和导航入口。
- `log.md`：按时间追加的操作记录。

## 权限与确认

- 读取和分析可以直接进行。
- 创建或更新少量 Wiki 页面前，先列出目标文件和理由。
- 批量写入、删除、移动、重命名、覆盖或修改本文件前，展示计划或 diff，等待用户确认。
- 用户未明确授权时，不访问 Vault 外部位置，不联网补充资料。
- 不把密钥、账号、私密路径或未脱敏个人信息写入共享文件。

## 证据规则

- 区分“来源明确说明”“根据来源推断”“当前未知”。
- Source 页用 `source_file` 记录 `raw/` 下的相对路径。
- 重要事实链接到 Source 页；能定位页码、章节、时间戳时同时记录。
- 新来源与旧结论冲突时保留两方证据，标记冲突，不静默覆盖。
- 不确定内容使用 `status: unverified` 或明确的待确认措辞。

## 页面与链接

- 一个页面聚焦一个可复用主题，避免为了数量过度拆分。
- 新页面至少连接一个已有页面；暂时无法连接时在 `index.md` 标注待整理。
- 使用 Obsidian wikilink 连接 Vault 内页面，外部地址使用 Markdown 链接。
- 页面 frontmatter 至少包含 `type`、`status`、`created`、`updated`。
- 文件名稳定、可搜索；改名时同步更新链接和索引。

## 工作流

### 摄取

1. 读取本文件和 `index.md`。
2. 分析一个或一小批新来源。
3. 先给出创建/更新计划。
4. 确认后创建 Source 页并更新相关 Wiki 页面。
5. 更新 `index.md`，向 `log.md` 追加记录。
6. 验证 `raw/` 未变化、链接可解析、结论可追溯。

长文档先按 `_guides/long-document-extraction.md` 建立提取清单，在 `_staging/` 中解析和分块；禁止一次提示词直接处理整本书。每批完成后记录范围、遗漏和下一批起点。

### 查询

1. 先查 `index.md`，再读取相关 Wiki 和 Source 页。
2. 回答中标明证据页面；证据不足时缩小结论。
3. 只有当回答具有长期复用价值时，才建议写回 Wiki。
4. 写回前说明目标文件并等待确认。

### 巡检

- 检查断链、孤立页、重复概念、冲突结论、陈旧摘要和未核实项。
- 先输出报告和最小修复方案；修复前确认。

## 场景规则

## 场景规则：项目管理

### 目标

从会议、邮件和文档中提取已确认的决策、任务、风险和里程碑。

### 页面类型

- `meeting`：会议来源、参会者、议题和记录。
- `decision`：背景、选项、决定、负责人和日期。
- `task`：动作、owner、due、status 和来源。
- `risk`：触发条件、概率、影响和应对。
- `milestone`：可验收的阶段结果。

目录映射：Meeting 放 `wiki/sources/`，Task、Risk、Milestone 放 `wiki/topics/`，Decision 放 `wiki/synthesis/`。

### 创建条件

- 只有明确承诺的行动才创建 Task；讨论中的建议标为 proposal。
- 不猜测 owner、截止时间或完成状态，缺失字段标待确认。
- Decision 必须来自明确决定，而不是讨论倾向。
- 状态变化应有会议、消息或用户确认作为依据。

### 查询输出

- 状态报告区分已完成、进行中、阻塞和未知。
- 每项任务显示 owner、due 和证据来源。
- 冲突的会议记录并列展示，等待负责人确认。

### 额外检查

- 逾期任务不会被自动改成完成。
- 决策链接回会议或来源。
- 日志不替代正式 Task/Decision 页面。

## 完成检查

- `raw/` 没有变化。
- 新增事实可回到 Source 页和原文件。
- `index.md` 与实际页面一致。
- `log.md` 只追加，没有改写历史。
- 没有把推断写成确定事实。
