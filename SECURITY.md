# 安全策略

## 支持范围

安全修复只面向最新的 `main` 分支和最新 GitHub Release。旧版本中的文档错误或脚本问题，请先在最新版本复现。

## 私下报告漏洞

如果问题可能导致文件被覆盖或删除、隐私资料泄露、命令注入或凭据暴露，请不要创建公开 Issue。

优先使用仓库 **Security** 页面中的 **Report a vulnerability** 私下提交报告。如果该入口尚未启用，请通过维护者 GitHub 资料中的私下联系方式报告，并提供：

- 受影响的脚本、规则或版本
- 最小复现步骤
- 可能影响的文件和数据
- 已知的临时规避方式

请不要附带真实密钥或完整私人 Vault。维护者确认问题后会协调披露时间，并在修复发布后致谢报告者（除非报告者希望匿名）。

## 使用者责任

- 在临时 Vault 或版本控制下测试自动化流程。
- 不把密钥、令牌和私人路径写入共享的 `CLAUDE.md`。
- 批量写入、覆盖、移动或删除前检查 Agent 展示的计划。
- 根据所选 Agent 和模型提供方的隐私政策决定是否处理敏感资料。
